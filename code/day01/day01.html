2、js的历史背景

布莱登.艾奇
JavaScript诞生于1995年。起初它的主要目的是处理以前由服务器端负责的一些表单验证。在那个绝大多数用户都在使用调制解调器上网的时代，用户填写完一个表单点击提交，需要等待几十秒，完了服务器反馈给你说某个地方填错了......在当时如果能在客户端完成一些基本的验证绝对是令人兴奋的。当时走在技术革新最前沿的Netscape（网景）公司，决定着手开发一种客户端语言，用来处理这种装简单的验证。当时就职于Netscape公司的布兰登·艾奇开始着手计划将1995年2月发布的LiveScript同时在浏览器和服务器中使用。为了赶在发布日期前完成LiveScript的开发，Netscape与Sun公司成立了一个开发联盟。而此时，Netscape为了搭上媒体热炒Java的顺风车，临时把LiveScript改名为JavaScript，所以从本质上来说JavaScript和Java没什么关系。
95 -- 网景      liveScript       
java              javascript

h5  和 js  关系   ：   是一样的  目的都是让静止的东西动起来

17、关系运算符

运算符 ：  >    >=   <  <=   == （等于）     ===(全等)    !=（不等于）        !== （不全等）  
表达式 ：
    5>6        5===6      
表达式的值 ： 
    关系表达式结果是一个布尔值   true 或 false
    如果关系表达式成立，结果为true   否则为false  

比较规律 ： 
    不能正常比较的结果都是false
    如果运算符中有一侧是纯数字字符串  自动类型转换后正常比较
    如果运算符两侧都是数值  正常比较
    如果运算符两侧都是字符串  ，比较字符串中某个字符的 ASCII码值
     a --   97     A  -- 65    0 ---- 48   

== 和 === 区别 ：
 ==  只比较值  
 === 先比较类型  再比较值

18、逻辑运算符

运算符 ：   && 与       ||或         ！非
表达式 ：
    5>6 && 6==7     !false        5&&6    5||6
表达式的值 ： 
   第一种情况 ：
        运算符的两侧都是布尔值 ： 
         && （且） ：   两侧如果都是真，结果为真true        true && true ===== true  
         ||     （或） ：  两侧如果都为假，结果为假 false       false || false ======false  
         !  (非) ： 非真即假   非假 即真    !false ====    true
    第二种情况（短路）：
        如果 false  碰到  && ,  &&右侧的表达式不执行
        如果 true  碰到了  ||   ， ||  右侧的表达式不执行
    第三种情况 ： 表达式中可能有一侧 为具体的值   
        4 || 3    4
        0 || 3     3   
        true && 5   5
        0 && 3   0

    几种数值情况 ：  0 表示假      非0表示 真     

19、条件运算符 (三目运算符)

表达式 ：
    5>6 ? 45  : 34
    表达式1 ?   表达式2  ：  表达式 3
表达式的值 ：
     如果表达式1为真，取  表达式2的值作为 整个条件表达式的值  
    如果表达式1位假  ， 取  表达式3 的值为整个条件表达式的值
